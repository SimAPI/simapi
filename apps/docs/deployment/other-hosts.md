# Other Hosts

SimAPI runs on any platform that supports **Node.js 20+**. All of these run a persistent Node.js server process, so SQLite logging and all SimAPI features work without modification.

| Host             | Notes                                                        |
| ---------------- | ------------------------------------------------------------ |
| **Render**       | Web Service — build: `npm run build`, start: `npm run start` |
| **Fly.io**       | `fly launch` with the included Dockerfile                    |
| **Heroku**       | Add a `Procfile`: `web: npm run start`                       |
| **VPS / Docker** | `docker build` + `docker run -p 3000:3000`                   |

---

## Render

1. Push your project to GitHub.
2. Go to [render.com](https://render.com) → **New Web Service** → connect your repo.
3. Set:
   - **Build Command**: `npm run build`
   - **Start Command**: `npm run start`
4. Add environment variables in the Render dashboard.

Render sets `PORT` automatically. SimAPI reads it and binds to the correct port.

---

## Fly.io

With a Dockerfile (generated by `simapi init`):

```sh
fly launch   # configures fly.toml from your Dockerfile
fly deploy
```

Set secrets with:

```sh
fly secrets set SIMAPI_CONSOLE_USERNAME=admin SIMAPI_CONSOLE_PASSWORD=secret
```

---

## Heroku

Add a `Procfile` at the project root:

```
web: npm run start
```

Then deploy:

```sh
heroku create
heroku buildpacks:set heroku/nodejs
git push heroku main
```

Set environment variables with:

```sh
heroku config:set SIMAPI_CONSOLE_USERNAME=admin SIMAPI_CONSOLE_PASSWORD=secret
```

---

## VPS / Docker

Build and run the Docker image:

```sh
docker build -t my-api .
docker run -p 3000:3000 \
  -e SIMAPI_CONSOLE_USERNAME=admin \
  -e SIMAPI_CONSOLE_PASSWORD=secret \
  my-api
```

Or with `docker compose`:

```yaml
services:
  api:
    build: .
    ports:
      - "3000:3000"
    environment:
      SIMAPI_CONSOLE_USERNAME: admin
      SIMAPI_CONSOLE_PASSWORD: secret
```

## Environment variables

| Variable                  | Effect                                              |
| ------------------------- | --------------------------------------------------- |
| `PORT`                    | Overrides the port from `simapi.config.ts`          |
| `SIMAPI_CONSOLE_USERNAME` | HTTP Basic Auth username for `/__simapi/*`          |
| `SIMAPI_CONSOLE_PASSWORD` | HTTP Basic Auth password for `/__simapi/*`          |
| `DATABASE_URL`            | Postgres connection URL (if using Postgres logging) |
| `TURSO_TOKEN`             | Auth token for libSQL/Turso logging                 |
